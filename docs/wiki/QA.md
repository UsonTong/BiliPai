# QA 测试手册

最后更新：2026-05-08（按当前 main 分支功能、隐私与发布文档校对）

## 目标

这份手册用于发版前和日常回归。重点不是把所有页面点一遍，而是覆盖真实用户最容易感知的问题：

- 核心功能是否可用：播放、搜索、空间、动态、番剧、直播、离线缓存、插件、更新。
- 状态是否可信：登录态、画质、播放进度、下载进度、插件启停、设置持久化。
- 隐私是否克制：登录凭证只在本地使用，权限弹窗有明确功能原因，遥测和插件能力可被用户理解与控制。
- 文档是否同步：README、CHANGELOG、Wiki 和版本号不互相打架。

## 测试环境建议

- Android 版本：至少覆盖 Android 10、12、14；有条件时补 Android 15/16 预览系统。
- 设备形态：手机 + 平板或折叠屏；至少覆盖竖屏、横屏、分屏或大屏布局。
- 网络：Wi-Fi、移动网络、弱网、代理或 DNS 异常场景。
- 账号：未登录、已登录非大会员、已登录大会员。
- 构建：优先使用同一 release candidate APK；不要混用旧包和当前源码判断问题。

## 发布前必测

### 1) 启动、导航与基础视觉

步骤：
1. 冷启动应用，观察启动页、首页首屏、底栏和顶部栏。
2. 在首页、搜索、动态、个人空间、设置之间快速切换。
3. 切换浅色/深色/动态取色、Material 3 Expressive、液态玻璃、底栏悬浮和贴底模式。
4. 在手机竖屏、手机横屏、平板/折叠屏宽屏下重复一次主导航。

期望：
- 首屏不长期空白，底栏、顶部栏、壁纸/动态取色不会互相遮挡。
- 底栏动画、指示器、搜索胶囊和液态玻璃在切换时不明显跳动或重叠。
- 平板/折叠屏不误用手机布局，设置页和主内容区不出现双重底栏。

### 2) 播放、画质、弹幕与 PiP

步骤：
1. 未登录、非大会员、大会员三种状态分别打开普通视频。
2. 检查首播清晰度、手动切换清晰度、DASH/MP4 回退、倍速、弹幕开关和弹幕过滤。
3. 进入横屏和竖屏全屏，测试拖动进度、长按倍速、暂停后恢复、返回详情页。
4. 进入 PiP，再用系统 PiP 按钮执行播放/暂停；随后回到应用继续播放。
5. 锁屏或切后台，检查后台播放、媒体通知、上一首/下一首/播放暂停控制。

期望：
- 非会员不会错误展示不可用画质；大会员高画质可按账号权限展示。
- 暂停、恢复、PiP、通知控制不会因为状态滞后反向触发。
- 弹幕设置即时生效，不影响播放器主链路。
- 横竖屏手势区域不和控制层、弹幕层、进度条互相抢事件。

### 3) 番剧、直播与听视频

步骤：
1. 打开番剧详情，切换季度/版本/选集，测试追番、点赞、投币等状态展示。
2. 进入直播分区和直播间，验证 HLS 起播、直播弹幕、返回和重新进入。
3. 从收藏夹、稍后再看、视频详情进入听视频，切换顺序播放、单曲循环、列表循环。
4. 切后台和锁屏，验证听视频封面、标题、媒体中心和播放队列。

期望：
- 番剧选集、播放进度和互动状态不串集。
- 直播间未登录能力降级可理解，不应卡死在半加载态。
- 听视频显示的封面、标题、当前播放项和系统媒体通知一致。

### 4) 首页、搜索、空间与动态

步骤：
1. 首页刷新、滚动、返回顶部，观察卡片复用、播放量/弹幕/时长显示。
2. 搜索视频、UP 主、番剧，检查历史记录、实时建议和结果跳转。
3. 进入 UP 空间，切换主页、投稿、动态、收藏等分区；在投稿视频中切换网格/单列布局。
4. 动态页快速切换 UP、刷新、打开图片/GIF 预览、返回列表。

期望：
- 首页卡片不重复 key 崩溃，不出现明显空白、错位或统计胶囊被挤成省略号。
- 搜索结果与空间页跳转稳定，返回后列表位置合理。
- 空间投稿布局切换平滑，单列/网格都能进入播放。
- 动态图片预览和 GIF 不长时间卡住，异常可重试恢复。

### 5) 评论、消息、历史与收藏

步骤：
1. 打开普通视频评论区和楼中楼，切换最热/最新。
2. 长按评论进入可选择复制，检查富文本、链接和特殊标签。
3. 打开消息中心，检查分类、已读状态和跳转。
4. 打开历史、收藏夹、稍后再看，测试单条删除、批量删除、播放跳转。

期望：
- 评论排序、楼中楼加载、复制面板和富文本链接都可用。
- 历史和收藏操作不误删其他列表，删除后 UI 与本地状态一致。
- 消息跳转不破坏当前导航栈。

### 6) 离线缓存、图片保存与 WebDAV

步骤：
1. 对同一视频选择不同清晰度缓存，观察任务创建、进度、暂停/继续、失败重试。
2. 网络中断后恢复，检查下载任务是否能继续或明确失败。
3. 进入本地播放，验证音视频合并、标题、封面和播放进度。
4. 长按图片保存到相册，分别在 Android 9 及以下、Android 10+ 验证权限与保存结果。
5. 若启用 WebDAV 备份/恢复，使用测试目录验证连接、上传、下载和失败提示。

期望：
- 进度不出现异常百分比，失败原因可读。
- 离线文件和本地播放状态不污染在线播放记录。
- 图片保存只在必要系统版本触发存储权限。
- WebDAV 失败不会泄露账号信息到普通 UI 文案或可分享日志。

### 7) 插件与扩展能力

步骤：
1. 打开插件中心，逐一启停内置插件：空降助手、去广告、弹幕增强、夜间护眼、今日推荐单、CDN 属地优选。
2. 导入一个 JSON 规则插件，检查预览、安装、启停和删除。
3. 导入 `.bpplugin` 包，检查 manifest、SHA-256、签名状态和能力授权提示。
4. 启用含敏感能力的插件后，回到首页、播放页、弹幕页观察功能边界。

期望：
- 插件状态持久化，重启后仍符合用户选择。
- 第三方插件的 `NETWORK`、`LOCAL_HISTORY_READ`、`LOCAL_FEEDBACK_READ`、`PLAYER_CONTROL` 等能力必须在安装前可见。
- 外部 `.bpplugin` 当前只做解析/预览/授权提示，不应静默执行外部 Dex。
- 插件异常不应直接破坏首页、播放和弹幕主链路。

## 隐私与权限专项

### 登录与本地数据

检查项：
- 扫码登录后，退出登录应清理可见账号状态。
- 登录凭证只用于 Bilibili 接口访问，不出现在普通日志、崩溃摘要、分享文本或截图中。
- 本地播放历史、今日推荐反馈、插件本地配置可被清理或重置。
- 多账号相关功能未完成前，不要承诺账号间数据完全隔离。

### 系统权限

检查项：
- 网络权限只服务内容加载、播放、下载、插件导入、更新检查、WebDAV 和投屏。
- 通知权限只在后台播放、下载或播放通知需要时解释用途。
- 安装未知应用权限只在应用内更新安装 APK 时需要。
- `WRITE_EXTERNAL_STORAGE` 只覆盖 Android 9 及以下图片保存场景。
- 位置/附近设备/Wi-Fi 相关权限要和 DLNA/本地网络发现绑定，未使用投屏时不应主动打扰用户。
- 权限拒绝后功能应降级或给出明确提示，不能卡死。

### 遥测、崩溃与日志

检查项：
- 未放入 `google-services.json` 时，构建应能跳过 Firebase 能力。
- Crashlytics/Analytics 文案要和实际默认值一致：崩溃追踪、使用统计、播放器诊断日志分别说明，不混成一个开关。
- 调试日志不得包含 Cookie、access token、手机号、邮箱、精确位置、WebDAV 密码。
- 导出日志或反馈时，应优先包含版本、设备、网络、功能路径和错误码，而不是隐私明文。

### 第三方内容与插件

检查项：
- README、插件中心和导入弹窗都应提醒用户审阅第三方插件规则和能力声明。
- JSON 插件 URL 下载失败、格式错误、规则过宽时，错误提示应可读。
- CDN 属地优选、推荐过滤、弹幕过滤等插件要能独立关闭。
- 插件读取本地历史或反馈时，UI 必须明确这是本地处理，不应暗示上传到云端。

## 文档与发布一致性

发布前检查：
1. `app/build.gradle.kts` 的 `versionCode` / `versionName` 与 `CHANGELOG.md` 顶部版本一致。
2. `README.md`、`README_EN.md` 的版本徽章、最近更新和截图引用可在 GitHub 上渲染。
3. `docs/wiki/FEATURE_MATRIX.md`、`docs/wiki/ARCHITECTURE.md`、`docs/wiki/RELEASE_WORKFLOW.md` 如涉及同一能力，应同步更新或明确滞后。
4. 新增权限、插件能力、遥测行为、下载/安装流程变化时，必须同步更新本 QA 的隐私专项。
5. Release 文案不要只写最后一个修复，要覆盖本次从上个版本到当前的用户可感知变更。

## 自动化命令

按变更面选择最小验证集：

```bash
# 目标策略/结构测试，优先用于 UI 策略、播放器策略、插件规则等窄改动
./gradlew :app:testDebugUnitTest --tests '<ExactTestName>'

# App 逻辑和策略的较广回归
./gradlew :app:testDebugUnitTest

# Kotlin/Compose 编译验证
./gradlew :app:compileDebugKotlin

# 文档、README、版本号或截图引用改动
git diff --check
```

不需要每次都跑 `assembleDebug`。只有触及 Manifest、资源打包、签名、安装、应用内更新 APK、Release 产物或用户明确要求时，才进入打包/安装/真机 smoke。
