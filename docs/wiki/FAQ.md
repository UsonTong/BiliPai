# 用户常见问题

最后更新：2026-05-08（面向当前 main 分支用户能力整理）

这份问答面向普通用户，尽量用直白的话解释 BiliPai 能做什么、为什么需要某些权限、隐私数据怎么处理，以及遇到问题时该怎么排查。

## 基础使用

### Q：BiliPai 是什么？

A：BiliPai 是一个第三方 Bilibili Android 客户端，主要覆盖视频播放、番剧、直播、动态、搜索、个人空间、消息、离线缓存、插件和大屏体验。它不是官方客户端，也不会承诺覆盖官方 App 的所有入口。

### Q：支持哪些 Android 版本？

A：当前面向 Android 8.0+ 设备。Android 12+ 可以获得更完整的动态取色、系统媒体控制和部分现代视觉效果。

### Q：必须登录才能用吗？

A：不是。首页、搜索、公开视频和部分直播可以未登录使用。但高清画质、大会员画质、收藏、稍后再看、历史、评论互动、追番等能力通常需要登录或账号权限。

### Q：为什么有些视频只能看 720P 或更低？

A：画质由视频本身、账号登录状态、会员权益、Bilibili 接口返回和设备解码能力共同决定。未登录或非大会员场景下，部分清晰度不会开放；如果账号状态刚变化，可以退出视频页后重新进入。

### Q：平板和折叠屏能用吗？

A：可以。应用已经适配平板/折叠屏的侧栏、底栏、横屏播放和部分宽屏布局。不同 ROM 的分屏、旋转策略可能略有差异，出现布局异常时建议附带设备型号和系统版本反馈。

## 播放与缓存

### Q：支持后台播放和听视频吗？

A：支持。视频可以在后台继续播放，也可以进入听视频模式。锁屏或切后台后，系统媒体中心和通知栏应能控制播放、暂停、上一首和下一首。

### Q：支持画中画吗？

A：支持 Android 系统的 PiP。进入画中画后，可以使用系统提供的播放/暂停按钮；如果系统不显示按钮，先确认设备系统是否允许该应用使用画中画。

### Q：弹幕能过滤或高亮吗？

A：可以。内置弹幕增强能力支持关键词、用户 UID/hash 等规则，外部 JSON 规则插件也可以扩展弹幕过滤或高亮。规则过严可能导致弹幕明显变少，可以先关闭插件排查。

### Q：离线缓存失败怎么办？

A：先检查网络、账号登录状态、清晰度权限和存储空间。缓存链接可能会过期，重试时应用会尽量刷新下载地址。如果进度显示异常、一直失败或本地播放失败，反馈时请提供视频链接、目标清晰度、网络环境和错误提示。

### Q：图片保存为什么有时要权限？

A：Android 9 及以下保存图片到相册需要存储权限；Android 10+ 通常不需要旧式存储权限。拒绝权限后，保存图片功能可能不可用，但不应影响普通播放。

## 登录与账号

### Q：登录信息会上传到 BiliPai 服务器吗？

A：不会。登录凭证只保存在本地，用于访问 Bilibili 官方接口。项目没有自建账号服务器，也不需要把你的 Cookie 或 token 上传给第三方服务。

### Q：怎么登录？

A：当前只提供扫码登录。登录后可用能力仍取决于你的 Bilibili 账号权限，例如大会员画质、收藏、稍后再看、历史和互动能力。

### Q：退出登录会清掉所有本地数据吗？

A：退出登录应清掉可见账号状态和登录凭证，但本地缓存、设置、部分播放记录或插件配置可能仍保留。需要彻底清理时，可以结合应用内清理入口和系统应用设置。

### Q：多账号切换支持吗？

A：多账号切换仍属于计划能力。当前不要把它当成账号隔离方案使用；如果你经常切换账号，建议先退出当前账号再登录另一个账号。

## 外观与功能查找

### Q：外观可以自己调吗？

A：可以。设置里提供主题、动态取色、Material 3 Expressive、液态玻璃、底栏样式、动画、播放器、弹幕、后台播放和插件等选项。默认配置面向通用场景，如果你觉得动效、底栏、颜色或播放器手感不合适，建议先在设置里 DIY。

### Q：功能入口找不到怎么办？

A：优先使用设置页搜索和应用内已有分类查找。很多功能不是单独放在首页按钮里，而是在设置、播放面板、插件中心、空间页、缓存页或长按菜单中。确认找不到后，再去 README、Wiki、CHANGELOG 或已有 issue 里搜索关键词。

### Q：遇到问题要直接发 issue 吗？

A：先自己做一次基础排查：确认版本是否最新、设置搜索是否能找到相关开关、插件是否影响行为、README/Wiki/CHANGELOG/已有 issue 是否已经解释。仍然无法解决时，再提交 issue，并带上版本、设备、系统、复现步骤和截图/录屏。

## 插件与扩展

### Q：插件是做什么的？

A：插件用于扩展推荐过滤、弹幕过滤/高亮、空降助手、夜间护眼、今日推荐单、CDN 属地优选等能力。内置插件随应用发布，外部 JSON 插件可以通过 URL 导入。

### Q：第三方插件安全吗？

A：导入前需要自己审阅规则和能力声明。尤其要关注 `NETWORK`、`LOCAL_HISTORY_READ`、`LOCAL_FEEDBACK_READ`、`PLAYER_CONTROL` 等敏感能力。来源不明的插件不要随便启用。

### Q：`.bpplugin` 能直接运行外部代码吗？

A：当前外部 `.bpplugin` 主要用于包格式、manifest、签名、SHA-256 和能力授权预览，不应被理解为已经稳定执行外部 Dex 的成熟插件生态。真正影响应用行为的外部轻量插件主要是 JSON 规则插件。

### Q：开启插件后播放或首页异常怎么办？

A：先进入插件中心逐个关闭最近启用的插件，再重启应用或刷新页面。如果关闭插件后恢复，反馈时请附上插件名称、规则来源、启用能力和复现页面。

## 隐私与权限

### Q：应用会收集我的隐私数据吗？

A：BiliPai 的原则是本地优先和权限克制。登录凭证、本地历史、推荐反馈、插件配置等主要保存在本地。使用 Bilibili 服务时，应用会按功能需要访问 Bilibili 官方接口。

### Q：为什么需要通知权限？

A：通知权限用于后台播放、下载任务和媒体控制提示。拒绝通知权限后，普通前台播放仍应可用，但后台播放控制和下载提示可能受影响。

### Q：为什么需要安装未知应用权限？

A：只有使用应用内更新并安装下载好的 APK 时才需要。你也可以选择手动去 GitHub Releases 下载和安装。

### Q：为什么会出现位置、附近设备或 Wi-Fi 相关权限？

A：这些权限主要和 DLNA/本地网络发现、投屏设备扫描有关。未使用投屏时，不应该频繁打扰你授权；拒绝后投屏能力可能降级，但普通播放不应受影响。

### Q：Crashlytics 和 Analytics 是什么？

A：Crashlytics 用于崩溃追踪，Analytics 用于使用情况统计。它们是否可用取决于构建是否放入 `google-services.json` 以及应用设置。反馈问题时，播放器诊断日志和普通调试日志应避免包含 Cookie、token、密码等明文隐私。

### Q：可以完全离线使用吗？

A：不能完全离线。在线播放、搜索、动态、直播、登录、插件导入和更新检查都需要网络。已经缓存完成的视频可以在本地播放，但封面、字幕、弹幕或账号状态仍可能需要网络才能补齐。

## 更新与反馈

### Q：怎么确认我正在用最新版本？

A：以 GitHub Releases、应用内更新检查和 `CHANGELOG.md` 为准。README 和 Wiki 会定期维护，但主分支变化很快，遇到不一致时优先看最新 release 和源码版本号。

### Q：应用内更新失败怎么办？

A：检查网络、GitHub 访问情况、存储空间和“安装未知应用”权限。也可以直接打开 GitHub Releases 手动下载 APK。

### Q：反馈问题时需要提供什么？

A：先用设置搜索、README、Wiki、CHANGELOG 和已有 issue 自查。确认不是设置项、插件或已知问题后，再提供版本号、设备型号、Android 版本、账号状态、网络环境、复现步骤、截图或录屏、相关错误提示。如果是播放、下载、登录、插件、隐私权限问题，请说明具体视频/插件/权限场景。

### Q：哪些问题适合优先反馈？

A：崩溃、无法起播、登录态异常、画质明显错误、缓存损坏、隐私权限文案不清楚、插件能力提示缺失、平板/折叠屏严重破版，以及 release 文档与实际行为不一致的问题最值得优先反馈。
